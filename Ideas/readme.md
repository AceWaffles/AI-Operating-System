
# Ideas

Purpose:

Preserve useful ideas without turning every idea into active work.

Core rule:

Every idea must answer one question:

When does this matter?

## Lifecycle

### Seeds

Raw ideas that are worth preserving but not ready for planning.

Use Seeds when:

- The idea is interesting but timing is unclear.
- Dependencies are missing.
- The idea needs more shaping before it can become work.
- The idea should not distract from current priorities.

### Saplings

Ideas with enough shape to revisit later.

Use Saplings when:

- The purpose is clear.
- The owner is known.
- Dependencies are understood.
- The idea may become useful this season or in a future season.

Saplings are not commitments.

### Active Work

Active work belongs in `Now.md`, roadmaps, or project files.

An idea becomes active only when it supports the current season, milestone, or an approved task.

### Completed Work

Completed work belongs in Morning Meetings, project summaries, or roadmaps.

Do not rewrite old history. Add a new summary when completed work changes current understanding.

### Parked Or Rejected Ideas

If an idea is intentionally abandoned, mark it clearly in its file instead of deleting it.

Reason matters because it prevents the same idea from being re-litigated later.

## Daily Evaluation

When Michael asks what to build today, AIOS should evaluate ideas against:

- Current season
- Current priorities
- Active blockers
- Technical debt
- Current milestone
- Next milestone
- Saplings that have become feasible
- Available time
- Risk of distraction
- Expected value

AIOS should recommend the highest-value task, not simply continue the most recent conversation.
